# Day 16 接口系统测试记录

## 测试目标

验证项目主要 HTTP 接口的路由、统一响应、参数校验与角色权限，确保公开内容、患者功能和后台功能遵循既定接口契约。

## 测试方式

- 使用 Spring Boot `MockMvc` 加载 Controller 和完整 Spring Security 过滤链。
- 使用 Mock 替代 Service 与 Mapper，测试过程不会修改本机 MySQL 数据。
- 使用 Maven 执行全部后端测试，确认接口测试与已有 Service 测试可以共同通过。

## 覆盖范围

| 模块 | 验证内容 |
| --- | --- |
| 认证 | 患者登录、注册参数校验、匿名访问 |
| 公开内容 | 知识文章列表、药膳列表 |
| 患者功能 | 创建问诊、我的问诊、AI 问答 |
| 权限 | 匿名请求受保护接口、患者访问后台接口 |
| 后台统计 | Dashboard 统计响应 |
| 内容管理 | 知识文章与药膳的查询、新增、更新、删除 |
| 数据导出 | CSV 响应类型、文件名与内容 |

### 测试中发现的问题

- 项目原先缺少 `spring-security-test` 测试依赖，已补充为 test scope。
- MyBatis Mapper 扫描会进入 MVC 测试上下文，接口测试中已使用 Mock 隔离全部 Mapper，避免连接或修改真实数据库。
- 未发现需要修改业务代码的接口错误。
